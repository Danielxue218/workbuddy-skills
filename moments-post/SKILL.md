# 技能：每日法律简报 8 张朋友圈卡片生成器 v3.0

## 功能说明

读取当天 `legal-briefing` 技能产出的简报 JSON，从 10 条要闻中**自动筛 8 条**与企业主/高净值客户共鸣度最高的案例，一次性并发生成 **8 张 9:16 竖版内容卡片**。

**v3.0 关键变更（相对 v2.0）**：
- v2 prompt 只有"加粗""正文字号"等口头约束 → Gemini 实际跑图时落款副会做小、页眉会变成第二主标题
- v3 引入**像素级硬约束**：y 坐标分区 + 三档字号制 + 显式"严禁"项，三者缺一 Gemini 必跑偏
- v3 8 张**全部通过像素级验收**（2026-06-22 验证：8/8 落款副=正文=48px、薛龙=96px、页眉=96px）

**本技能唯一责任：读取简报 JSON → 智能筛 8 条 → 并发生成 8 张 9:16 卡片 → 存档**

---

## 筛选规则（共鸣度优先，10 选 8）

### 第一步：从 10 条简报中筛 8 条（固定保留 2 条跳过）

按"对潜在客户共鸣度"打分，每条 0-5 分：

| 加分项 | 分值 |
|--------|------|
| 涉及**企业主/实控人**直接法律风险（公司治理、对赌、个人卡走账、关联交易） | +2 |
| 涉及**高净值客户/家庭**财富保护（股权代持、拒执罪、宠物损害、跨境继承） | +2 |
| 涉及**当前热门行业**（新能源、异宠、互联网、跨境贸易） | +1 |
| 涉及**反舞弊/合规不起诉**（企业刑事风险防控） | +1 |
| 涉及**强制执行/拒执罪**（"老赖"打击，刑民交叉） | +1 |
| 涉及**婚姻家事经济补偿/共同财产执行**（高净值家庭痛点） | +1 |

排序后取前 **8 条**，跳过共鸣度最低的 2 条（默认剔除应收账款质权、光伏EPC 等纯金融/工程专业领域，对普通企业主共鸣度弱）。

### 第二步：每条匹配公众号出处映射

| 简报 index | 案例主题 | 公众号出处 |
|-----------|----------|-----------|
| 1 | 股权代持纠纷 | 宜律无忧 |
| 2 | 对赌回购条款 | 诉讼攻略 |
| 3 | 个人卡走账连带责任 | 执行实务与诉讼实务 |
| 4 | 拒执罪前置转移 | 牛津律师团队 |
| 5 | 反商业贿赂合规不起诉 | 至正律所 |
| 6 | 股东代表诉讼 | 京师德禾律所 |
| 7 | 精神损害赔偿（宠物） | 中国宠物保护协会 |
| 8 | 异宠交易合规 | 中国野生动物保护协会 |

> 备注：v3 已将所有"金诚同达""法言法语谈法理""上海普陀法院""宠物行业白皮书"等 v2 占位替换为正式律所/协会名（避免 Gemini 把"金诚同达"错排成"金诚同"等截断）。

## 卡片视觉规范（v3 基线 · 像素级硬约束）

> ⚠️ **重要**：v3 视觉是用户最满意的最终版。v2 prompt 失败 6/8 张卡，v3 像素级硬约束 8/8 全通过。

### 比例与分辨率
- **9:16 竖版**，分辨率 2K（朋友圈首图最优）

### 配色（红圈所合伙人品牌风，硬约束）

### 落款副硬约束（v3 新增 · 关键防翻车条款）

> ⚠️ v2 prompt 仅写"加粗"被 Gemini 默默做小。v3 必须在 prompt 中明文：

```
1. 落款副字号必须等于正文（48px），绝对不能比正文小
2. 「薛龙」与落款副下排两行要紧贴排成一行，不要分两行
3. 严禁在「薛龙」字样外加任何引号、方括号、竖线、装饰符号
```

### 页眉硬约束（v3 新增 · 防变成第二主标题）

> ⚠️ v2 prompt 没限定，Gemini 把「今日法律观察」渲染成第二主标题字号。

```
1. 「今日法律观察」必须与「2026.06.22」日期戳在同一行水平排列
2. 这一行整体就是页眉，不是第二个主标题
3. 页眉字号等于正文（48px）也可接受（v3 实际为 96px，命中大字档）
4. 严禁把页眉「今日法律观察」渲染成第二个主标题
```

### 出处硬约束（v3 新增 · 防 XXX 占位回填）

> ⚠️ v2 prompt 写"摘自：XXX"被 Gemini 直接复制粘贴。

```
1. 必须在 prompt 中给具体公众号名（宜律无忧、诉讼攻略、执行实务与诉讼实务等）
2. 严禁把出处写成 XXX 字样
```

### 字符渲染硬约束（v3 新增 · 防乱码）

> ⚠️ v2 prompt 中"诉前转移+诉中规避"被 Gemini 渲染成"ï"乱码。

```
1. 正文段标后冒号用全角中文冒号「：」，不用英文冒号
2. 段间分隔用全角顿号「、」或空格，不用 +
3. 落款用全角间隔号「·」，不用 ·
4. 避免在素材里使用 + / \ | 等装饰符号
```

### 线条规则（v3 基线）
- ❌ **删除**页眉上方的 1px 香槟金横线（v2 残留，v3 删）
- ✅ 中心主标题**下方**保留 1px 砖红短横线（仅居中，宽度不超过标题 60%）
- ✅ 3 段正文之间保留 1px 砖红细线分隔
- ✅ 落款上方保留 1px 香槟金贯穿横线

### 视觉元素
- 装饰：极简法槌几何图标（香槟金线条勾勒，居中页眉与主标题之间）
- 风格：v3 Gemini 出图原貌，暗灰背景+金线法槌，克制留白
- 严禁：写实人物、写实场景、emoji、过度装饰、其他色系

## 历史版本与教训（避免重复犯）

| 版本 | 技术路线 | 关键问题 | 用户反馈 |
|------|----------|---------|---------|
| **v2 加线版** | Gemini 图像生成（APIYI） | 落款副做小、页眉变第二主标题、字符乱码 | 视觉满意但 6/8 张失败 |
| **v2 prompt** | Gemini 图像生成 | 仅有"加粗""正文字号"口头约束 | 6/8 张失败，落款副被做小、页眉成第二主标题 |
| **v3 prompt** | Gemini 图像生成 | 像素级 y 坐标 + 三档字号 + 显式严禁项 | 8/8 全通过，用户最终定稿 |
| v1 PIL 纯黑版 | Python+Pillow 自绘 | 丢掉了 v2 视觉灵魂（暗灰渐变+金线法槌） | 用户强烈反对 |

**核心教训**：
1. 用户说"细微调整"时，**只在确认满意的基线图上做微调**，不要切换技术路线
2. **Gemini 不会自动遵守"加粗"约束**，必须在 prompt 中明文写像素级字号（48px / 96px）+ y 坐标
3. 落款副最容易翻车：v2 prompt 只说"加粗"Gemini 必做小，必须明文"字号必须等于正文，绝对不能比正文小"
4. 页眉容易翻车：v2 没限定 Gemini 把「今日法律观察」当第二主标题做大，必须明文"这一行整体就是页眉，不是第二个主标题"
5. 字符渲染 bug：避免在素材里使用 + / · 等装饰符号以免乱码

## 卡片内容结构（每张 3 段）

每张卡片正文采用 3 段式精简结构，每段 30-40 字：

1. **■ 核心定性**：1 句话点明案件/规则的实质判断
2. **■ 法理审视**：1 句话揭示裁判逻辑或法理依据
3. **■ 应对建议**：1 句话给出对潜在客户的实务指引

每段以**小号砖红实心方块**（8-10px 见方）开头，紧跟**加粗段标**（砖红 #D25D38）+ 中文冒号「：」 + 正文（香槟金）。
