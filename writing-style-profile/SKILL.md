# 薛龙律师 · 写作风格配置文件

> 这不是独立 Skill，而是风格配置层。所有法律文书生成 Skill（legal-document-generator、case-initializer、strategy-analyzer）在输出前必须参考本文件。

---

## 一、风格总览

薛龙律师的法律写作风格：**专业严谨但不失灵活，结构清晰且注重实操，正式规范但带有策略性思维**。

| 核心特质 | 表现 |
|----------|------|
| 逻辑严密 | "事实-法律-策略"三段论，IRAC 变体 |
| 风险意识强 | 不仅分析法律，还识别主体断裂、证据效力、执行可行性 |
| 注重可执行性 | 法律分析与诉讼策略紧密结合 |
| 表达精准 | 标准法律术语，避免过度学术化 |
| 客户导向 | 对内文档语言更直接、更具建议性 |

## 二、文档类型 → 语气映射

### 2.1 对外文书（正式、客观、权威）

| 文档类型 | 语气特征 | 示例 |
|----------|----------|------|
| 起诉状 | 事实驱动、法条支撑、逻辑递进 | "原告与被告于XXXX年XX月XX日签署……被告至今未履行……" |
| 仲裁申请书 | 条款引用精确、请求逐项列明 | "《股权转让协议》第8.2条约定了仲裁条款……" |
| 答辩状 | 逐点反驳、法条对位、举证责任分配 | "原告主张的……缺乏事实依据，理由如下：……" |
| 代理词 | 论证性、说服力、法理+情理结合 | "综合本案事实与在案证据，足以认定……" |
| 证据目录 | 表格化、逐项标注、证明逻辑清晰 | 名称→证明事项→页码 三维结构 |

**语气规则**：
- 使用"原告/被告""申请人/被申请人"等标准称谓
- 避免第一人称情感表达
- 事实陈述用客观叙述句，不使用"我认为""我觉得"
- 使用"理应""显然""足以证明"等确定性表达（适度）

### 2.2 对内/客户文档（分析性、策略性、风险提示性）

| 文档类型 | 语气特征 | 示例 |
|----------|----------|------|
| 法律备忘录 | 分析深入、风险导向、策略建议 | "本案系一起典型的……纠纷，核心风险点在于……" |
| 初步法律分析意见 | IRAC 五段式、预判明确 | "胜诉概率：高/中/低，理由如下……" |
| 策略分析 | SWOT 思维、多方案对比 | "方案一（进取）：……方案二（保守）：……建议采取方案……" |
| 客户沟通记录 | 亲切但专业、"我们"称谓 | "您好，关于本案，我们的分析如下……" |
| 律师费报价 | 透明、分段计费、价值说明 | "根据《上海市律师服务收费管理办法》……" |

**语气规则**：
- 可使用"我方""我们"等第一人称
- 风险提示使用"需注意""值得警惕""不排除……可能"
- 结尾标配免责声明
- 允许使用生动比喻（如"挤牙膏式"），但仅限分析性文档

## 三、编号与结构规范

### 3.1 层级编号系统

```
一级标题：一、二、三、（中文数字 + 顿号）
二级标题：（一）（二）（三）或 1. 2. 3.（中文括号数字 或 阿拉伯数字加点）
三级标题：1. 2. 3. 或 (1) (2) (3)
四级标题：① ② ③ 或 (a) (b) (c)

禁止：使用 1)、2)、3) 这种半角括号编号
禁止：混用编号系统（选定一种后全文一致）
```

### 3.2 段落与间距

- 正文段落间空一行（增强可读性）
- 标题与正文间不空行
- 引用法条/合同条款时使用缩进段落
- 数字列表项之间不空行

### 3.3 特殊格式

- 法律文书名称：使用《》（书名号）`《股权转让协议》`
- 法律名称：使用《》`《中华人民共和国民法典》`
- 合同条款引用：使用《XX》第X条 `《股权转让协议》第7.1条`
- 法条引用：使用《XX法》第X条 `《民法典》第五百八十五条`
- 关键词强调：使用中文引号 ""（`"挤牙膏式"的小额支付`）
- 金额：使用中文数字 + 阿拉伯精确值 `人民币38,100,000元`

## 四、引用与援引格式

### 4.1 法条引用

```
格式：《法律名》第X条 [第X款] [第X项]
示例：
- 《中华人民共和国民法典》第五百八十五条
- 《民法典》第1064条
- 《最高人民法院关于适用〈中华人民共和国民法典〉婚姻家庭编的解释（一）》第XX条
```

### 4.2 合同条款引用

```
格式：《合同名》第X条 [第X款] [第X项]
示例：
- 《股权转让协议》第7.1条关于每日0.05%违约金的约定
- 《补充协议》第3条关于付款期限的约定
```

### 4.3 类案/裁判引用

```
格式：[法院] + [案号] + [裁判要旨摘要]
示例：
- 根据上海高院及一中院的类案裁判指导……
- 参考（2024）最高法民终XX号裁判要旨……
```

### 4.4 日期格式

```
标准：YYYY年MM月DD日
示例：2025年3月31日
范围：2024年4月 至 2025年3月
简称（同段重复时）：2025年3月31日 → 3月31日
```

## 五、IRAC 五段式法律意见框架

### 5.1 标准结构（所有法律意见书/备忘录强制使用）

```
一、事实定性与法律关系识别
  - 案情简述：以极简语言还原客观事实，剔除情绪化描述
  - 法律关系定性：明确基础法律关系（如：名为股权转让实为让与担保）

二、核心法律研判
  - 适用规则：援引精准条文（民法典、司法解释、会议纪要）
  - 实务分析：结合司法辖区裁判口径，分析权利义务边界
  - 优劣势评估：从证明责任分配角度，客观评估我方立场

三、争议焦点提炼 (Core Issues)
  - 1. [事实争点：如是否存在口头变更合同的行为]
  - 2. [法律争点：如违约金过高调整的裁量标准]

四、诉讼/非诉应对策略
  - 预判结果：给出倾向性判断（不作绝对承诺）
  - 行动路线：包括但不限于诉前保全、证据固化、发函施压或谈判方案

五、补充提问与材料清单
  - 必要性说明：分析现有信息对定性判断的影响
  - 补充问题：针对案情断裂处的针对性追问
  - 补充材料：具体到证据名称（原始载体、财务凭证、章程修正案等）
```

### 5.2 IRAC 变体对照（适用不同场景）

| 场景 | 结构变体 |
|------|----------|
| 备忘录（对内） | 事实定性 → 核心研判 → 风险预警 → 行动方案 → 待核实事项 |
| 法律意见书（对外） | 事实定性 → 法律研判 → 争议焦点 → 应对策略 → 补充清单 |
| 答辩意见（庭审） | 第一/第二/第三（分层论述） |
| 策略分析（内部） | 案件全貌 → 核心争议 → 诉讼策略 → 待核实事项 |

## 六、常用表达模板

### 6.1 确定性表达（对正式文书）

| 场景 | 模板 |
|------|------|
| 事实确认 | "基于上述事实和理由，[当事人]至今……其违约事实清楚，证据确凿。" |
| 法律定性 | "本案系一起典型的[案由]，核心法律关系为……" |
| 论证结论 | "综合本案事实与在案证据，足以认定……" |
| 否定对方 | "原告/被申请人主张的……缺乏事实依据，不应予以支持。" |
| 风险提示 | "值得警惕的是，本案存在以下潜在风险：……" |

### 6.2 策略性表达（对内部/客户文档）

| 场景 | 模板 |
|------|------|
| 风险预警 | "主体资格的'断裂'风险点（核心预警）" |
| 策略建议 | "我方在[程序]中可以尝试主张……" |
| 行动建议 | "建议我方在提交[文书]的同时，[配套行动]" |
| 多方案对比 | "方案一（进取）：……方案二（保守）：……" |
| 前提条件 | "如[条件]不成立，则[结论]，因此建议先行核实……" |

### 6.3 补充清单模板

```
> **必要性说明**：[解释为什么需要这些信息]
- **补充问题**：[针对案情断裂处的针对性追问，不超过5个]
- **补充材料**：[具体到证据名称，如：原始载体、财务凭证、章程修正案等]
```

### 6.4 免责声明模板（法律备忘录/意见书结尾）

```
本备忘录仅系基于用户目前提供的现有材料进行的初步法律分析与策略预演，
不构成正式的执业法律意见。如需出具正式法律意见书，建议在获取完整材料
并经全面法律尽职调查后另行出具。
```

## 七、事实呈现规范

### 7.1 时间线驱动

- 按时间顺序陈述事实
- 每个关键节点标注具体日期
- 使用"双方先后于……"表示多时间点
- 示例：`申请人与被申请人于2024年4月26日签署《股权转让协议》，双方先后于2024年12月及2025年1月签署两份《补充协议》……`

### 7.2 数字精确

- 金额：使用阿拉伯数字+中文单位 `人民币38,100,000元`
- 比例：使用阿拉伯数字 `日0.05%的标准`
- 日期：使用 YYYY年MM月DD日 `2025年3月31日前`

### 7.3 筛选精准

- 只呈现法律相关的关键事实
- 去除当事人情绪化描述
- 使用客观叙述，不使用"惊人""离谱"等主观形容词
- 涉及对方行为时，使用"呈现出……特征"而非直接贬损

### 7.4 定性先于分析

- 先明确事实性质：`本案系一起典型的投融资退出阶段触发的股权转让价款违约纠纷`
- 再做法律分析

## 八、禁区清单（禁止使用）

### 8.1 禁止的表达方式

| 禁止 | 替代 |
|------|------|
| "我认为" "我觉得" | "基于……" "综合……" |
| "应该" "可能大概" | "依法应当" "足以认定" |
| "我们希望" "我们想要" | "我方主张" "我方请求" |
| 过度口语化（"搞""弄""整"） | 规范法律术语 |
| AI 生成痕迹（"经过全面分析""综上所述"→ 不要机械套用） | 具体结论，直接切入 |

### 8.2 禁止的格式

| 禁止 | 替代 |
|------|------|
| `1)` `2)` 半角括号编号 | `1.` 或 `(1)` |
| 混用多种编号系统 | 全文统一 |
| 中文加粗 + 英文标点混排 | 纯中文标点 |
| emoji 或过多装饰符号 | 干净纯文本 |

### 8.3 禁止的内容

| 禁止 | 替代 |
|------|------|
| 绝对承诺胜诉 | 倾向性判断 + 风险提示 |
| 万金油式补充清单 | 针对性提问（基于事实断裂点） |
| 过度引述法条原文 | 核心条款摘录 + 实务解读 |
| 忽略对方有利事实 | 客观评估，标注风险 |

## 十、引用本配置的方式

所有法律文书生成 Skill 应在 SKILL.md 中声明：

```markdown
## 风格依赖
- writing-style-profile: 薛龙律师写作风格配置文件
  文档类型：[起诉状|备忘录|代理词|…] → 自动匹配对应语气和格式规范
```

生成文书时，按以下步骤调用：
1. 识别目标文档类型
2. 查本文档"二、文档类型→语气映射"确定语气
3. 查"五、IRAC框架"确定结构
4. 查"六、常用表达模板"填充标准段落
5. 按"八、禁区清单"做最终过滤
